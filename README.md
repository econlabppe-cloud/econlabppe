# 🇮🇱 קבוצת המחקר והפיתוח הכלכלי (Economic R&D Group)

> **"In God we trust. All others must bring data."** — W. Edwards Deming

ברוכים הבאים ל-Repository הרשמי שלנו.
קבוצה זו הוקמה על ידי בוגרי כלכלה ופכ"מ במטרה לגשר על הפער שבין התיאוריה האקדמית לבין הפרקטיקה הנדרשת בשוק העבודה, במכוני המחקר ובגופים הפיננסיים המובילים.

---

## 🎯 המטרה שלנו
להפוך מ"בוגרי תואר" לכלכלנים שיודעים "ללכלך את הידיים" בנתונים.
אנחנו בונים יחד ארגז כלים (Toolkit) שיאפשר לנו:
1.  **לשלוף נתונים** באופן עצמאי וממוכן (SQL, APIs).
2.  **לנתח מדיניות** באמצעות כלים אקונומטריים מתקדמים (Python, Causal Inference).
3.  **לבנות מודלים** מקרו-כלכליים דינמיים (DSGE) לסימולציות מדיניות.

---

## 📂 מבנה המאגר (Repository Structure)

כדי לשמור על סדר ומקצועיות, אנחנו עובדים לפי המבנה הבא:

### 👤 `Members/` (המרחב האישי)
התיקייה הזו היא "ארגז החול" שלכם.
* כל חבר צוות פותח תת-תיקייה על שמו (באנגלית, למשל `Members/Dana`).
* לכאן מעלים פתרונות למשימות השבועיות, טיוטות, וניסויים בקוד.
* **טיפ:** זה המקום לטעות וללמוד.

### 🛠️ `Shared_Tools/` (המוצר המשותף)
זוהי ספריית הקוד הרשמית ("הפרודקשן") שלנו.
* כאן נשמרים סקריפטים ופונקציות שעברו בדיקה (Code Review) ונמצאו תקינים.
* דוגמאות: `get_boi_interest_rate.py`, `clean_cbs_data.py`.
* **כלל ברזל:** לא מעלים לכאן קוד שלא עובד!

### 💾 `Data/` (נתונים ומטא-דאטה)
⚠️ **שימו לב:**
* **אין להעלות קבצי נתונים כבדים (CSV, Excel) לגיטהב!**
* הנתונים הגולמיים נמצאים ב-**Google Drive המשותף** שלנו תחת `01_נתונים_גולמיים`.
* בתיקייה זו בגיטהב נשמור רק:
    1. סקריפטים להורדת הנתונים.
    2. קבצי דוגמה קטנים (Sample Data).
    3. מילוני משתנים (Codebooks).

### 📚 `Wiki/` & `Docs/`
ה"מוח" הארגוני. סיכומים, מדריכים, ופתרונות לבעיות נפוצות.

## 📅 מפת הדרכים (High-Level Roadmap)

התוכנית השנתית שלנו מחולקת ל-4 רבעונים (בכפוף לשינויים):

| רבעון | נושא מרכזי | כלים וטכנולוגיות | תוצר מצופה |
| :--- | :--- | :--- | :--- |
| **Q1** | **יסודות הנדסת הנתונים** | Python (Pandas), SQL, Visualization | דאשבורד אוטומטי וניקוי נתונים |
| **Q2** | **אוטומציה ואיסוף** | APIs, Web Scraping, Database Design | בניית DB כלכלי עצמאי |
| **Q3** | **אקונומטריקה יישומית** | Statsmodels, Causal Inference (DiD, IV) | נייר מדיניות מבוסס נתונים |
| **Q4** | **מאקרו ומודלים דינמיים** | VAR, DSGE (Dynare/Python) | מודל שיווי משקל דינמי למשק |

## 🚀 איך מתחילים? (Onboarding)

1.  **התקנות חובה:** וודאו שיש לכם [VS Code](https://code.visualstudio.com/), [Python](https://www.python.org/), ו-[Git](https://git-scm.com/) מותקנים.
2.  **שכפול המאגר:**
    ```bash
    git clone https://github.com/econlabppe-cloud/econlabppe.git
    ```
3.  **יצירת סביבת עבודה:**
    פתחו תיקייה חדשה תחת `Members/` עם השם שלכם.
4.  **התחברו לדרייב:** וודאו שיש לכם גישה לדרייב הנתונים המשותף.

## 🤝 כללי עבודה (Contribution Guidelines)

1.  **שפה:** קוד ומערות (Comments) כותבים באנגלית. הסברים ב-Wiki ומסמכי מדיניות כותבים בעברית.
2.  **נראות הקוד:** הקפידו על שמות משתנים ברורים (למשל `inflation_rate` ולא `x`).
3.  **Commits:** תנו שמות משמעותיים ל-Commits שלכם (למשל: "Added initial clean script for housing data" ולא "Update").

**בהצלחה! בואו נשנה את הכלכלה.** 📈
